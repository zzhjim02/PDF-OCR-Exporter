## 简介

PDF OCR提取工具是一个专门用于从双层PDF文件中提取OCR识别文本的图形化应用程序。它支持批量处理、保持目录结构、文件名过滤等功能。

### 主要特性
- 递归扫描文件夹中的所有PDF文件
- 提取PDF文件中的OCR文本层内容
- 保持原有的文件名和目录结构
- 支持文件名过滤(通配符匹配)
- 实时显示处理进度
- 详细的错误日志记录
- 简洁友好的图形界面

## 安装

### 系统要求
- Python 3.8 或更高版本
- Windows / Linux / macOS

### 安装步骤

1. **下载项目**
   ```bash
   git clone <repository-url>
   cd pdf-ocr-extract
   ```

2. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```

3. **运行程序**
   ```bash
   python main.py
   ```

## 快速开始

### 基本使用流程

1. **启动程序**
   运行 `python main.py` 启动应用程序

2. **选择源目录**
   点击"源目录"后的"浏览..."按钮,选择包含PDF文件的文件夹

3. **选择输出目录**
   点击"输出目录"后的"浏览..."按钮,选择保存TXT文件的文件夹

4. **(可选)设置过滤条件**
   在"文件名过滤"输入框中输入通配符模式,例如:
   - `*.pdf` - 所有PDF文件(默认)
   - `report_*.pdf` - 以report_开头的PDF文件
   - `*.pdf` - 所有PDF文件

5. **开始提取**
   点击"开始提取"按钮,程序将自动处理所有PDF文件

6. **查看结果**
   处理完成后,可以在"处理结果"标签页查看统计信息
   在"错误日志"标签页查看详细的处理日志

## 功能说明

### 1. 目录选择
- **源目录**: 包含待处理PDF文件的文件夹
- **输出目录**: 保存提取的TXT文件的文件夹

### 2. 文件名过滤
支持通配符匹配:
- `*` 匹配任意数量的字符
- `?` 匹配单个字符
- 示例: `*.pdf`, `report_*.pdf`, `file?.pdf`

### 3. 覆盖选项
- **不勾选**: 跳过已存在的TXT文件
- **勾选**: 覆盖已存在的TXT文件

### 4. 进度显示
- 实时显示处理进度(当前/总数)
- 显示当前正在处理的文件名
- 显示处理状态信息

### 5. 处理结果
- 总文件数
- 成功处理的文件数
- 失败的文件数
- 跳过的文件数
- 处理耗时

### 6. 错误日志
- 记录所有处理失败的文件及其错误信息
- 支持清除和保存日志功能

## 使用场景

### 场景1: 批量提取文档文本
将一个文件夹中的所有PDF文档的文本提取出来,便于后续的文本分析和处理。

### 场景2: 保持目录结构
从复杂的多层目录结构中提取PDF文本,输出时保持原有的目录结构。

### 场景3: 选择性处理
使用文件名过滤功能,只处理特定命名规则的PDF文件。

### 场景4: 增量更新
不勾选"覆盖已存在的文件"选项,只处理新增的PDF文件。

## 常见问题

### Q1: 程序提示"未安装PDF处理库"怎么办?
A: 确保已安装所有依赖:
```bash
pip install -r requirements.txt
```

### Q2: 提取的文本为空或乱码怎么办?
A: 可能的原因:
- PDF文件不包含OCR文本层(仅包含图像)
- PDF文件的编码格式特殊
- 建议检查PDF文件是否为双层PDF格式

### Q3: 可以处理加密的PDF文件吗?
A: 当前版本不支持加密PDF文件,需要先解密。

### Q4: 处理速度慢怎么办?
A: 处理速度取决于:
- PDF文件的大小和数量
- 计算机的性能
- 建议分批处理大量文件

### Q5: 如何查看详细的错误信息?
A: 在"错误日志"标签页查看,或保存日志文件到本地。
