# 多线程和跳过功能使用指南

## 概述

PDF OCR提取工具 v1.1.0 新增了多线程并行处理和智能跳过已处理文件功能,可以显著提升大批量文件的处理效率。

---

## 多线程处理功能

### 什么是多线程处理?

多线程处理允许程序同时处理多个PDF文件,而不是逐个顺序处理。这样可以充分利用多核CPU的性能,大幅提升处理速度。

### 使用方法

1. **启用多线程**
   - 在主界面中勾选"启用多线程处理"复选框(默认已勾选)

2. **设置线程数**
   - 使用"线程数"选择器设置并发线程数
   - 可选范围: 1-16个线程
   - 默认值: 4个线程

3. **开始处理**
   - 点击"开始提取"按钮
   - 程序将使用指定数量的线程并行处理PDF文件

### 线程数选择建议

| CPU核心数 | 推荐线程数 | 说明 |
|----------|-----------|------|
| 2核 | 2-3线程 | 适合低配置电脑 |
| 4核 | 4-6线程 | 适合主流配置 |
| 8核 | 8-12线程 | 适合高性能电脑 |
| 16核+ | 12-16线程 | 适合服务器级配置 |

**注意事项**:
- 线程数不是越多越好,过多的线程可能导致系统资源争抢
- 建议从默认值开始,根据实际效果调整
- 如果遇到内存不足,可以减少线程数

### 性能对比

假设处理100个PDF文件:

- **单线程模式**: 约100秒(每个文件1秒)
- **4线程模式**: 约25-30秒(提升3-4倍)
- **8线程模式**: 约15-20秒(提升5-6倍)

*实际性能取决于文件大小、CPU性能和磁盘I/O速度*

## 跳过已处理文件功能

### 什么是跳过已处理文件?

启用此功能后,程序会自动检查输出目录中是否已存在对应的TXT文件。如果存在,则跳过该PDF文件的处理,避免重复工作。

### 使用方法

1. **启用跳过功能**
   - 在主界面中勾选"跳过已处理的文件"复选框(默认已勾选)

2. **开始处理**
   - 点击"开始提取"按钮
   - 程序会自动跳过已存在的TXT文件

3. **查看结果**
   - 在"处理结果"中可以看到"跳过"的文件数量
   - 在"错误日志"中会显示每个跳过的文件

### 使用场景

#### 场景1: 增量处理
```
第一次处理: 100个PDF文件 -> 100个TXT文件
第二次处理: 新增10个PDF文件 -> 只处理10个新文件
```

#### 场景2: 断点续传
```
处理到第50个文件时程序崩溃
重新启动程序,会自动跳过前49个已处理的文件
从第50个文件继续处理
```

#### 场景3: 定期更新
```
定期扫描源文件夹,只处理新增的PDF文件
避免重复处理已有文件
```

### 与"覆盖已存在的文件"的关系

| 跳过已处理 | 覆盖已存在 | 行为 |
|-----------|-----------|------|
| ✓ | ✗ | 跳过已存在的文件(默认) |
| ✓ | ✓ | 覆盖已存在的文件 |
| ✗ | ✗ | 跳过已存在的文件 |
| ✗ | ✓ | 覆盖已存在的文件 |

**建议配置**:
- 日常使用: 勾选"跳过已处理",不勾选"覆盖已存在"
- 重新处理所有文件: 取消勾选"跳过已处理"或勾选"覆盖已存在"

## 常见问题

### Q1: 多线程处理会占用很多内存吗?
A: 是的,多线程处理会增加内存使用。如果遇到内存不足,可以减少线程数或使用单线程模式。

### Q2: 跳过功能如何判断文件已处理?
A: 程序会检查输出目录中是否存在与PDF文件同名的TXT文件(仅扩展名不同)。

### Q3: 多线程处理会影响文件顺序吗?
A: 不会影响最终结果,但处理顺序可能与单线程不同。进度显示会根据完成顺序更新。

### Q4: 如何查看哪些文件被跳过了?
A: 在"错误日志"标签页中,每个跳过的文件都会显示"文件已存在,跳过"的提示。

### Q5: 多线程处理速度提升不明显?
A: 可能的原因:
- PDF文件数量较少(少于10个)
- 文件非常大,受限于磁盘I/O
- 线程数设置不当
- CPU性能瓶颈

建议: 对于少量文件,单线程模式可能更高效。

## 技术细节

### 多线程实现
- 使用Python的`concurrent.futures.ThreadPoolExecutor`
- 每个PDF文件在一个独立的线程中处理
- 使用信号槽机制实现线程间通信
- 支持任务取消和优雅停止

### 跳过机制
- 在处理前检查输出文件是否存在
- 使用`TextFileWriter.exists()`方法
- 不影响其他文件的正常处理
- 跳过的文件会计入"跳过数"统计

## 性能优化建议

1. **合理设置线程数**
   - 根据CPU核心数和文件数量调整
   - 避免设置过多线程导致资源争抢

2. **使用SSD存储**
   - SSD可以显著提升文件读写速度
   - 特别是对于大量小文件的处理

3. **保持足够的内存**
   - 多线程处理会增加内存使用
   - 建议至少8GB内存

4. **关闭不必要的程序**
   - 处理大量文件时关闭其他占用资源的程序
   - 释放CPU和内存资源
